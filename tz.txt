You are the lead software engineer for this repository.

Your task is to implement a Windows desktop application called "Vocal Joystick" according to the technical specification stored in a text file in the root of the project.

You must treat that technical specification file as the source of truth for product requirements, architecture, constraints, and acceptance criteria.

## Product goal
Build a Windows 10/11 desktop application that allows a user with disabilities to control the mouse using vocal sounds.

The application has 7 actions:
- MoveUp
- MoveDown
- MoveLeft
- MoveRight
- LeftClick
- RightClick
- DoubleClick

Directional actions are activated by sustained vocal sounds and controlled using pitch-based detection and hold logic.
Click actions are activated by short trained sound patterns.

The application must support:
1. Configuration mode
   - User records and assigns sound patterns to each action
   - User can re-record, save, and test patterns
2. Working mode
   - Application continuously listens to microphone input
   - Recognizes trained sounds in real time
   - Moves the mouse and performs clicks

## Mandatory technology choices
- Language: C#
- Runtime: .NET 8
- UI: WPF
- Architecture: MVVM-like layered architecture
- Audio capture: NAudio
- Mouse control: Win32 SendInput
- Persistence: JSON under %AppData%/VocalJoystick/
- Tests: unit tests for core logic

## Important implementation constraints
- Windows only
- No cloud services
- No speech-to-text
- No online APIs
- No neural networks / heavy ML in MVP
- Everything must work locally
- Prefer clean, maintainable, production-style code
- Use nullable enabled
- Use dependency injection
- Avoid god classes
- Avoid hardcoded paths
- Keep responsibilities separated
